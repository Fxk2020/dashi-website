# 📚 读书笔记快速指南

## 🚀 如何添加新书籍（3分钟完成）

### 步骤 1：创建文件

在 `src/content/books/` 目录下创建新的 `.md` 文件：

```bash
# 文件名使用英文，用连字符分隔
src/content/books/your-book-name.md
```

**命名示例：**

- `nanming-history.md` ✅ （南明史）
- `deep-work.md` ✅
- `南明史.md` ❌（不要用中文）

---

### 步骤 2：复制模板

复制下面的模板到新文件中，然后填写内容：

```yaml
---
title: '书名'                        # 必填
author: '作者'                       # 必填
publisher: '出版社'                  # 可选
pubYear: '2024'                     # 可选
rating: 5                           # 可选，0-5，支持小数
readDate: '2024-11-28'              # 必填，格式 YYYY-MM-DD
tags: ['标签1', '标签2', '标签3']    # 可选
category: '分类'                     # 可选，如：历史、技术、小说
status: 'finished'                  # 可选：finished/reading/abandoned
summary: '一句话总结这本书的核心内容。'  # 必填
thoughts: |                         # 必填，读后感想（多行）
  第一段：为什么读这本书 / 初读印象
  
  第二段：核心观点 / 最震撼的内容
  
  第三段：印象深刻的金句（可用引号）
  
  第四段：对我的启发 / 实践收获
  
  第五段：总结 / 推荐
---
```

---

### 步骤 3：实际示例

这是《南明史》的完整示例：

```yaml
---
title: '南明史'
author: '顾诚'
publisher: '中华书局'
pubYear: '2011'
rating: 5
readDate: '2024-11-28'
tags: ['历史', '明清史', '南明']
category: '历史'
status: 'finished'
summary: '顾诚先生关于南明历史的扛鼎之作，详细记录了明朝灭亡后南方政权的兴衰历程。'
thoughts: |
  这是我读过的最详实、最客观的南明史著作。顾诚先生用扎实的史料和严谨的考证，还原了明朝灭亡后十八年间南明政权的真实面貌。

  书中最让我震撼的是对南明失败原因的分析。不是简单归咎于"党争"或"内耗"，而是深入剖析了当时的政治、军事、经济等多重因素。

  "明之亡，亡于党争；南明之亡，亦亡于党争" —— 但顾诚先生指出，这只是表象。真正的问题是整个统治阶层已经无法适应时代变革。

  印象最深的是史可法守扬州、李定国抗清等片段。这些民族英雄在绝境中的坚守让人动容，但个人英雄主义终究无法挽救一个王朝的倾覆。

  读完这本书，我对明清交替这段历史有了全新的认识。历史不是简单的黑白对错，而是无数因素交织的复杂结果。

  这本书文笔流畅，考证严谨，既有学术价值，又有可读性。强烈推荐给所有对明清史、南明历史感兴趣的朋友。
---
```

## 📋 字段说明速查

### 必填字段（5个）

| 字段         | 说明     | 示例                |
| ------------ | -------- | ------------------- |
| `title`    | 书名     | `'南明史'`        |
| `author`   | 作者     | `'顾诚'`          |
| `readDate` | 阅读日期 | `'2024-11-28'`    |
| `summary`  | 简短总结 | `'一句话总结...'` |
| `thoughts` | 读后感想 | 见模板              |

### 可选字段

| 字段          | 说明     | 示例                         |
| ------------- | -------- | ---------------------------- |
| `publisher` | 出版社   | `'中华书局'`               |
| `pubYear`   | 出版年份 | `'2011'`                   |
| `rating`    | 评分     | `5` 或 `4.5`             |
| `tags`      | 标签数组 | `['历史', '明清史']`       |
| `category`  | 分类     | `'历史'`                   |
| `status`    | 状态     | `'finished'`/`'reading'` |

## ✍️ 读后感想编写技巧

### 建议结构（5段式）

```
第1段：为什么读这本书 / 初读印象（50-100字）
第2段：核心观点 / 最震撼的内容（100-150字）
第3段：印象深刻的金句（可用 "引号" 或 —— 破折号）
第4段：对我的启发 / 实践收获（100-150字）
第5段：总结 / 推荐（50-100字）
```

### 写作要点

- ✅ 真实感受，不要流于表面
- ✅ 具体例子，不要空泛议论
- ✅ 金句引用，增加可读性
- ✅ 个人实践，分享收获
- ✅ 适度长度：建议 200-500 字

## ❓ 常见问题

**Q: thoughts 字段中可以分段吗？**
A: 可以！段落之间空一行即可。

**Q: 标签可以有多少个？**
A: 没有限制，但建议 2-4 个，页面只显示前 2 个。

**Q: 可以不写 publisher 和 pubYear 吗？**
A: 可以，这些是可选字段。

**Q: rating 可以不写吗？**
A: 可以，但建议填写，会显示星级评分。

**Q: 如何删除一本书？**
A: 直接删除对应的 `.md` 文件即可。

**Q: 修改后需要重启吗？**
A: 开发模式下会自动热更新，生产环境需要重新构建。

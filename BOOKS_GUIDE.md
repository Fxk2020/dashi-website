# 📚 读书笔记使用指南

本指南将帮助你快速上手添加和管理读书笔记。

## 📁 文件结构

```

├── the-three-body-problem.md    # 《三体》读书笔记
├── sapiens.md                   # 《人类简史》读书笔记
└── clean-code.md                # 《代码整洁之道》读书笔记
```

## ✍️ 如何添加新的读书笔记

### 步骤 1：创建 Markdown 文件

在 `src/content/books/` 目录下创建一个新的 `.md` 文件，文件名使用英文（会作为识别ID）。

例如：`src/content/books/deep-work.md`

### 步骤 2：添加 Frontmatter（元数据）

在文件开头添加以下格式的元数据（两个 `---` 之间）：

```yaml
---
title: '深度工作'                    # 书名（必填）
author: '卡尔·纽波特'                # 作者（必填）
publisher: '江西人民出版社'          # 出版社（可选）
pubYear: '2017'                     # 出版年份（可选）
rating: 4.5                         # 评分 0-5（可选）
readDate: '2024-05-15'              # 阅读日期（必填，格式：YYYY-MM-DD）
tags: ['个人成长', '时间管理', '效率'] # 标签（可选）
category: '个人成长'                 # 分类（可选）
status: 'finished'                  # 阅读状态：finished/reading/abandoned（可选）
summary: '一本教你如何在分心时代保持专注的书。' # 简短总结（必填）
thoughts: |                         # 读后感想（必填，使用 | 表示多行文本）
  这本书让我意识到专注的重要性...
  
  通过实践书中的方法，我的工作效率提升了50%...
  
  强烈推荐给所有想提升效率的朋友。
---
```

**注意**：`thoughts` 字段使用 `|` 符号表示多行文本，每段之间可以空一行。

## 📝 完整示例

```markdown
---
title: '深度工作'
author: '卡尔·纽波特'
publisher: '江西人民出版社'
pubYear: '2017'
rating: 4.5
readDate: '2024-05-15'
tags: ['个人成长', '时间管理', '效率']
category: '个人成长'
status: 'finished'
summary: '在这个充满干扰的时代，深度工作能力成为了一种稀缺的超能力。'
thoughts: |
  作为一名开发者，我经常被各种消息、会议打断。这本书教会我如何在分心时代保持专注。

  书中提出的"深度工作"概念非常实用：在无干扰的状态下专注进行职业活动，使个人的认知能力达到极限。我开始每天早上9:00-11:00关闭所有通知，专注写代码。这两个小时的产出超过了之前一整天。

  "如果你不重视自己的时间，别人也不会重视" —— 这句话让我印象深刻。我开始主动安排每一分钟的时间，减少社交媒体使用，内心也更平静了。

  通过实践书中的方法三个月后，我的工作效率提升了至少50%，代码质量也有明显改善。虽然有些方法过于理想化，但整体理念非常实用。强烈推荐给所有想提升工作效率、找回专注力的朋友。
---
```

## 🎨 字段说明

### 必填字段

- `title`：书名
- `author`：作者
- `readDate`：阅读日期（格式：YYYY-MM-DD）
- `summary`：简短总结（1-2句话，会显示在卡片顶部）
- `thoughts`：读后感想（多行文本，会显示在卡片下方）

### 可选字段

- `publisher`：出版社
- `pubYear`：出版年份
- `rating`：评分（0-5 之间的数字，支持小数如 4.5）
- `tags`：标签数组，如 `['小说', '科幻', '中国文学']`
- `category`：分类，如 `'科幻小说'`、`'技术'`、`'人文社科'`
- `status`：阅读状态
  - `finished`：已读完
  - `reading`：在读中
  - `abandoned`：已弃读

## ✨ 读后感想编写技巧

### 1. 结构建议

```
第一段：为什么读这本书 / 初读印象
第二段：核心观点 / 最震撼的内容
第三段：印象深刻的金句（可以用引号）
第四段：对我的启发 / 实践收获
第五段：总结 / 推荐
```

### 2. 写作建议

- **真实感受**：写出真实的阅读体验，不要流于表面
- **具体例子**：用具体的例子说明书中观点对你的影响
- **金句引用**：引用书中精彩语句，用引号或破折号标注
- **个人实践**：分享你如何应用书中的方法
- **适度长度**：建议 200-500 字，3-5 段

### 3. 示例模板

```yaml
thoughts: |
  [为什么读这本书] 作为一名XXX，我经常遇到XXX问题。看到这本书的标题，我立刻被吸引了。

  [核心观点] 书中最震撼的观点是XXX。作者通过XXX的例子说明了XXX。这让我意识到XXX。

  [金句引用] "XXX" —— 这句话让我印象深刻。它提醒我XXX。

  [实践收获] 读完这本书后，我开始XXX。通过实践，我发现XXX。我的XXX提升了XX%。

  [总结] 这是一本XXX的书。推荐给所有想要XXX的朋友。
```

## 📊 页面功能

### 列表页 (`/books`)

- **统计卡片**：显示总书籍数、已读完数量、在读数量
- **分类筛选**：点击分类按钮可以筛选不同类别的书籍
- **书籍卡片**：显示书名、作者、评分、摘要、**读后感想**等信息
- **响应式布局**：在不同设备上都有良好的显示效果

**特点**：所有内容都在一个页面展示，无需跳转，方便快速浏览。

## 🎯 最佳实践

### 1. 命名规范

文件名使用小写英文，用连字符 `-` 分隔单词：
- ✅ `the-three-body-problem.md`
- ✅ `deep-work.md`
- ❌ `三体.md`
- ❌ `Deep Work.md`

### 2. 分类建议

**按类型分类：**
- 科幻小说、历史小说、推理小说
- 技术、编程、软件工程
- 历史、哲学、心理学
- 个人成长、时间管理、效率

### 3. 标签使用建议

每本书建议添加 2-4 个标签，用于快速识别主题：
- 技术类：Python、JavaScript、架构设计
- 文学类：科幻、推理、爱情
- 社科类：历史、哲学、经济
- 成长类：职场、创业、投资

### 4. 评分标准

- ⭐⭐⭐⭐⭐ (5.0)：必读经典，强烈推荐
- ⭐⭐⭐⭐½ (4.5)：非常优秀，值得阅读
- ⭐⭐⭐⭐ (4.0)：很好，推荐阅读
- ⭐⭐⭐½ (3.5)：不错，可以读读
- ⭐⭐⭐ (3.0)：一般，选择性阅读

## 🚀 发布流程

1. 创建/编辑 Markdown 文件
2. 填写完整的 frontmatter
3. 编写读后感想
4. 提交到 Git：
   ```bash
   git add src/content/books/
   git commit -m "Add: 新增《书名》读书笔记"
   git push
   ```
5. 网站会自动重新构建并发布

## 💡 技巧

### 快速开始

可以复制已有的读书笔记文件作为模板，修改内容即可：

```bash
cp src/content/books/the-three-body-problem.md src/content/books/new-book.md
```

然后编辑 `new-book.md`，修改所有字段内容。

### 多行文本格式

`thoughts` 字段使用 `|` 符号后，可以直接换行编写：

```yaml
thoughts: |
  第一段内容...
  
  第二段内容...
  
  第三段内容...
```

段落之间空一行会在页面上显示为分段。

### 草稿功能

如果还没读完，可以将 `status` 设置为 `reading`，表示正在阅读。

## ❓ 常见问题

**Q: 评分必须是整数吗？**
A: 不是，支持小数，如 `4.5`。

**Q: thoughts 字段可以很短吗？**
A: 可以，但建议至少 100 字，3 段以上，这样页面效果更好。

**Q: 如何删除一本书的笔记？**
A: 直接删除对应的 `.md` 文件即可。

**Q: 可以修改已发布的笔记吗？**
A: 可以，直接编辑 `.md` 文件，提交后会自动更新。

**Q: thoughts 字段太长怎么办？**
A: 页面会自动添加滚动条，不用担心。建议控制在 500 字以内效果最佳。

**Q: 一定要填写所有可选字段吗？**
A: 不需要，只需要填写必填字段即可。但建议至少填写 `rating`、`tags`、`category` 这三个，可以让页面更丰富。
